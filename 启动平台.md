## 🔧 故障排查（实战经验）

### 问题1: 端口被占用

**错误信息:**
```
bind: address already in use
```

**原因:** 8080端口已被其他程序占用

**解决方案A: 关闭占用端口的程序**
```bash
# 第1步: 查找占用端口的进程
lsof -i :8080

# 输出示例:
# COMMAND   PID USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
# go      12345 user    3u  IPv6 0x1234      0t0  TCP *:8080 (LISTEN)

# 第2步: 杀死进程
kill -9 12345  # 替换为实际的PID

# 第3步: 验证端口已释放
lsof -i :8080  # 应该没有输出
```

**解决方案B: 使用其他端口**
```bash
# 修改端口为3000
PORT=3000 go run main.go

# 然后访问: http://localhost:3000
```

**解决方案C: 一键清理脚本**
```bash
# 创建清理脚本
cat > kill_port.sh << 'EOF'
#!/bin/bash
PORT=${1:-8080}
PID=$(lsof -ti:$PORT)
if [ -n "$PID" ]; then
    kill -9 $PID
    echo "✅ 已关闭端口 $PORT 的进程（PID: $PID）"
else
    echo "✅ 端口 $PORT 未被占用"
fi
EOF

chmod +x kill_port.sh
./kill_port.sh 8080
```

## 🆘 紧急救援（快速解决问题）

### 服务无法启动

```bash
# 一键诊断脚本
cat > diagnose.sh << 'EOF'
#!/bin/bash
echo "🔍 开始诊断..."
echo ""

echo "1. 检查Go环境..."
if command -v go &> /dev/null; then
    echo "✅ Go已安装: $(go version)"
else
    echo "❌ Go未安装，请先安装Go"
    exit 1
fi

echo ""
echo "2. 检查端口8080..."
if lsof -i :8080 &> /dev/null; then
    echo "❌ 端口8080被占用:"
    lsof -i :8080
    echo "运行: kill -9 <PID> 关闭进程"
else
    echo "✅ 端口8080可用"
fi

echo ""
echo "3. 检查项目文件..."
if [ -f "main.go" ]; then
    echo "✅ main.go存在"
else
    echo "❌ main.go不存在，请检查目录"
    exit 1
fi

echo ""
echo "4. 检查笔记本文件..."
NOTEBOOK_COUNT=$(ls ../../*.ipynb 2>/dev/null | wc -l)
echo "✅ 找到 $NOTEBOOK_COUNT 个笔记本文件"

echo ""
echo "🎉 诊断完成！可以启动服务了。"
EOF

chmod +x diagnose.sh
./diagnose.sh
```

### 页面显示异常

```bash
# 一键修复脚本
cat > fix_display.sh << 'EOF'
#!/bin/bash
echo "🔧 开始修复显示问题..."

echo "1. 停止服务..."
pkill -f "go run main.go"
sleep 2

echo "2. 清理缓存..."
rm -rf /tmp/go-build*

echo "3. 重新启动..."
go run main.go &
sleep 3

echo "4. 测试服务..."
if curl -s http://localhost:8080 > /dev/null; then
    echo "✅ 服务正常"
    echo "📍 请在浏览器中强制刷新（Cmd+Shift+R）"
else
    echo "❌ 服务启动失败，请查看错误日志"
fi
EOF

chmod +x fix_display.sh
./fix_display.sh
```
